1.Vue.js文档学习

  • 生命周期函数：beforeCreated/ created/ boforeMount/ mounted/ beforeUpdate/ updated/ beforeDertroy/ destroyed/
    ! 不要在选项属性上或回调函数上使用箭头函数，因为箭头函数是和父级上下文绑定在一起的

  • V-html 可以输出真正的HTML，而双花括号会将数据解释为普通文本，所以需要注意：站点上动态渲染的HTML可能会比较危险，容易导致XSS攻击

  • v-on简写为@XXX="YYY" v-bind简写为:XXX="YYY"
  • Vue Router ($router 路由对象)
    • $router.path: 当前路由的路径
    • $router.params: key/value(object)
    • $router.query: URL查询参数、key/value(object)

Q: menu选中状态设置问题，当刷新时，路由因为设置跳转可以保持原路由，但是menu菜单的选中项会变成默认的设置项（default-active）
S:

将default-active设为当前的路径，而index设置为相应的路径,注意你之前使用的是go(‘path’)的方法设置跳转，建议不用，用来替代

S:

通过不同的时候的样式来进行切换，hover才显示的可以设置display：none/block来切换，而selected是通过vue绑定类名的方式来设置不同样式

！！！不能滥用watch,而是需要使用computed属性替代

计算属性
watch computed 两者均可以观察页面的数据变化，但是computed只在相关的数据发生变化的时候才会改变它的值，存在缓存，而watch每一次都会执行函数，所以要养成一个思想：computed属性要多用，watch不要滥用，目前还没看见该使用它的场合，有待研究

Q:点击列表上的按钮的时候，列表本身的click事件也会被触发
S:事件冒泡/阻止冒泡/事件捕获/阻止捕获相关知识

  • event.stopPropagation() 除IE外的浏览器使用
  • event.cancelBubble = true IE下才起作用，所以上述两个结合使用
  • 直接在click事件添加.stop：@click.stop = “changeState”
  • event.preventDefault(),不阻止事件冒泡，阻止默认行为
  • return flase的方法，阻止冒泡和默认

Q:点击整个页面除指定区域外的某个地方，改变一个状态地方法
S:

利用addEventListener添加全局点击事件，把指定区域用(id)选中，取反，注意
var _this = this 的this的绑定(理由暂时没研究出来，但是不这么写不会起作用)
this的指向问题
this 指向并不是在函数定义的时候确定的，而是在调用的时候确定的
函数的调用方式决定了 this 指向

  • 直接调用(window)、注意 bind() 的影响、call和 apply的影响（this指向传入函数的第一个参数）
  • 方法调用（指向调用该方法的对象）
  • 调用事件处理函数的时候，this 指向会绑定到触发事件的 DOM 元素上（可用var _this = this或bind或箭头函数来解决这一问题）
  • new调用（新创建的函数）
  • 箭头函数中的 this： 箭头函数没有自己的 this 绑定。箭头函数中使用的 this，其实是直接包含它的那个函数或函数表达式中的 this箭头函数是根据外层（函数或者全局）作用域来决定this
    箭头函数不会绑定this， 或则说箭头函数不会改变this本来的绑定，所以call()、apply()、bind()这些方法不会改变this的指向
