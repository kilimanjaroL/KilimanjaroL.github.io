文本溢出

前言：有的时候文字的长度超出给定的范围，就会用 … 来代替，这种场景还是挺常见的，之前没有自己了解过，这一次改一个bug的时候遇到了，于是知道了这种叫做：文本溢出

实现方法：

单行文本溢出：

  • `white-space:nowrap：规定段落中的文本不进行换行
  • text-overflow：规定当文本溢出包含元素时发生的事情
    • clip：直接裁剪掉,
    • elipsis：裁剪了的部分用一个省略号代替显示

多行文本溢出

1）webkit的CSS扩展属性实现(webkit内核适用）

CSS3 box-orient 属性
规定框的子元素应该被水平或垂直排列
box-orient: horizontal|vertical|inline-axis|block-axis|inherit;

  • horizontal：在水平行中从左向右排列子元素
  • vertical：从上向下垂直排列子元素
  • inline-axis：沿着行内轴来排列子元素（映射为 horizontal）
  • block-axis：沿着块轴来排列子元素（映射为 vertical）
  • inherit：应该从父元素继承 box-orient 属性的值

！目前没有浏览器支持 box-orient 属性
— Firefox 支持替代的 -moz-box-orient 属性
— Safari、Opera 以及 Chrome 支持替代的 -webkit-box-orient 属性

so 上面的方法只适用于webkit内核的浏览器（Safari、Opera 以及 Chrome…）
